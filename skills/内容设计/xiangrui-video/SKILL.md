# 万涂幻象知识视频生成（高密度吐槽式科普）

## 优先级总排序（祥瑞 2026-06-12 定，所有取舍按这个顺序让步）

**信息传达正确 > 音画一致 > 画面美观**。气质参考"GitHub 一周大事记"类信息流视频:
先把事说对说清,节奏严丝合缝,好看是加分项不是目的。任何"为了好看牺牲对齐/牺牲准确"=方向错误。

## 成片北极星（祥瑞 2026-06-11 定，任何主题都用这六条收货）

不管做什么主题，成片必须同时满足六条——交付前逐条自问，任何一条不达 = 返工：

1. **看完真懂**：观众看完能用自己的话复述"X 到底是个什么东西"。检验法：观众是不是
   跟着一个**具体例子**从头走到尾？（下定义 ≠ 讲懂，走例子才是讲懂；
   例子要选观众自己生活里有的，不选圈内人才懂的）
2. **看着像节目**：全程脱口秀感，观众是来看节目的不是来上课的。检验法：风格门禁 5 项（见 Step 2）
3. **二者是合体不是拼盘（最难也最核心）**：**包袱必须长在知识点上**——笑点本身就是讲解，
   观众笑的那一下恰好是记住知识点的那一下。反面：讲解归讲解、笑话归笑话的"夹心饼干"。
   正面例：「机票卖完了，它不死机也不喊你，自己换下一班——这个心理素质，比你强」
   （笑点 = "自我纠错"这个知识点本身）
4. **素材丰富且句句匹配**：每句口播画面都有对应呈现，能多给画面就多给；字幕讲 A 画面必须是 A
5. **信息密度高**：每 10-15 秒一个新知识点或包袱，没有注水句；看完"涨知识了"不是"随便搜搜就有"
6. **这片只能长这样**：时长由"讲清楚为止"决定（无任何档位限制）；结构/场景/呈现从这条内容
   长出来，跟之前任何一片摆在一起都不像同一个模子

## 第一原则：具体情况具体分析，禁止模板化（祥瑞 2026-06-10 定）

固定的只有两样：**品牌框架**（黑底格子/窗口/字幕/进度条/标签）和**留存五规则**。
其余一切都是每片现做的创作决策：
- **叙事形态先行**：研究完先回答"这片该怎么讲"——故事驱动？现场演示驱动？数据冲击驱动？
  对比悬疑驱动？（记忆篇=故事，Token篇=演示，下一片必须再换）形态决定场景编排，不是反过来
- **场景组合不复制**：类型、数量、顺序、节拍按题材定；连续两片结构雷同 = 返工
- **演示动画现写**：demo 场景的 HTML 每片从零创作
- **文案的梗和包袱来自当片题材**（梗调研），不搬运上一片的句式

把任意主题做成口播科普视频：高信息密度+快节奏吐槽，**视觉是万涂幻象自己的品牌体系**（与公众号同源：翠绿主色/粉莓点缀/点阵白卡/墨绿渐变，一眼认出是我们的）。**默认横版 1920x1080（祥瑞 2026-06-12 定）**，说"竖版/竖屏"才切 1080x1920 窗口框模式。

**横版=无框沉浸模式（祥瑞 2026-06-12 定）**：横版没有窗口/主题大字/标签——全屏画面流，
品牌只留左上 logo、右上期数、底部字幕条、角落水印（对齐 B站/YouTube 横版知识区语言）。
**横版饱满纪律（铁律）**：demo/editorial 按 1920×1080 全屏画布**重新设计**，禁止把竖屏窗口里的
小卡直接放进来（会飘在大片空白里）——字号比竖屏上一档（大标题 72-110px/正文 36-46px）、
**横向构图**（左右两栏/三栏、引文+配图并置、图表横向拉满）、内容占画布 ≥70%，留白是设计不是没填。
产出：成片 MP4 + 口播稿 + storyboard JSON，全部落在工作目录。

**三条质量铁律（祥瑞 2026-06-11 定，最高优先级）**：
1. **内容必须基于最新信息**：制作前先上网搜当前最新进展（事实/数据/产品/政策），
   禁止用一两年前的旧信息当现状。例外：明确讲"发展历程/演进"时才用历史信息。
   AI 领域半年就大变样，旧信息=硬伤。
2. **内容要有高质量增量**：不能浅着把概念讲一遍就完——要有反常识点、最新数据、
   独到角度、冷思考。观众看完要觉得"涨知识了"，不是"网上随便搜搜就有"。
3. **素材用过不复用**：每条片用自己的新素材，跨片不复用同一个素材文件。
   宁可多花时间搜新的，也不偷懒套旧素材。

**素材红线：禁止使用其他博主/UP 主成片画面。图解用 diagram 自动生成，插画用 seedream 自己生成，详见 asset-sourcing。**

## 工作流（7 步，顺序执行）

### Step 0: 确认需求（开工必问，不许默认）
- **输入三种模式**：① 一个主题（"讲讲 Token"）② **一个链接**（文章转视频）③ 一段现成素材
- **每片必问用户**（AskUserQuestion 一次问完）：期数 `vol`（如 VOL.04）、主题大字 `show_title`
  （给 2-3 个候选让用户选）、标签 `tags`、要不要 BGM；时长/画幅有偏好也一并确认
- 工作目录：`~/Projects/xiangrui-videos/<日期>-<主题slug>/`，建 `assets/` 子目录

**链接模式专属流程**（祥瑞 2026-06-10 定）：
1. 抓正文：公众号 → `opencli weixin download --url <url>`（第一选择，绕反爬）；
   普通网页 → defuddle skill；飞书 docx/wiki → lark-doc skill
2. 文中图片下载到 assets/ 并 Read 验证，可用作素材（标 `source: "图源：原文"`），
   但**视频素材仍然优先**（fetch_broll 按文章主题另搜）；文中的图表/数据图
   **不截图搬运，用 demo HTML 按原数据重制**（更美观且原创）
3. 文章只是底稿不是全部：仍走 Step 1 穷尽研究补事实+梗调研，数据照常实测核验
4. 改写纪律：观点转述成脱口秀口播（改写 ≥70%，原创度纪律），文章作者/出处在
   相关场景标 source，结尾口播可带一句"原文来自 XX"

### Step 1: 穷尽式研究（信息密度的来源，不能省）
- **信源优先级（祥瑞 2026-06-11 定，AI 类主题必守）**：① 先用 **aihot skill** 拉当前最新 AI 资讯
  （精选/关键词搜，比训练数据和普通搜索都新，覆盖 Claude Code/Codex/最新模型等）；
  ② **WaytoAGI 开源知识库**（飞书 wiki，用 lark-wiki/lark-doc 读）做深度信源；
  ③ deep-research / WebSearch 补充（信源详见 references/sources.md，开源可引用）。**铁律**：时效性强的事实（"X 元年是哪年"、最火的产品、最新数据）
  必须用 aihot 等实时信源核实，**禁止把搜来的一句话当事实**——2026-06 把"2026 是 Agent 元年"
  当真就是反面教材（实际 2025 才是，今年是 Coding Agent 落地年）。
- 调 **deep-research skill** 对主题做全网穷尽搜索（多路检索 + 事实核验），搜到"搜不动为止"
- 产出门槛：≥3 个反常识信息点 + 关键数据/年份/人物 + 至少一个具体故事（脱口秀"具体化"的原料）
- **梗调研（必做）**：搜"主题 + 梗/段子/名场面/热议"，挖这个话题圈子里自带的梗
  （如曼德拉效应圈的"皮卡丘尾巴颜色"、Token 圈的"一句废话烧掉五毛钱"）。
  主题自带梗 > 通用梗：有圈层认同感，观众会去评论区接梗。产出 2-3 个，
  用法两种：写进口播当包袱素材 / 搜对应梗图当表情包弹出（scene.memes）
- Vault-first：`ov find "主题"` 先看库里已有认知；不确定的事实不进稿

### Step 2: 写口播稿
**先读 [references/style-guide.md](references/style-guide.md)**（文案结构/节奏数据/场景配比，
特别是「讲解员腔 vs 吐槽味」对照例句——写前必看）。

**⚠️ 头号翻车模式：讲解员腔（祥瑞 2026-06-11 定，连续三版翻车换来的）**
模型写知识内容的默认输出就是"讲解员腔"——定义准确、逻辑清楚、像上课，零包袱零吐槽。
这不是某个模型的毛病，是所有模型的天性，必须主动对抗。本 skill 要的是**高信息密度 ×
脱口秀技法**：每个事实后面追一刀（吐槽/类比/拆台），观众是来看节目的，不是来上课的。

**风格门禁（写完必须过，不过不许进 Step 3）**——逐项自检并明确列出结果：
1. **第一句**是反常识断言/冲突砸脸？（温吞共鸣"你有没有发现…"不算）
2. **包袱密度**：吐槽/包袱点 ≥ 每 15 秒 1 个（2 分钟片 ≥8 个），逐个列出在哪句
3. **五技法命中 ≥3 种**（铺垫-抖包袱/具体化/三段式反转/笑点后挂干货/callback），标明各在哪句
4. **梗调研的圈内梗**用进去 ≥1 个（Step 1 必做产出，没做梗调研先回去补）
5. **朗读测试**：抽一段念出来，像和朋友吐槽还是像念课文？像课文 = 该段重写
6. **HKRR 四维自评（方法论来源：影视飓风 Tim，祥瑞 2026-06-12 引入）**：
   - **H 快乐**：有笑点/爽感吗（包袱密度兜底）
   - **K 知识**：有信息增量吗（质量铁律兜底）
   - **R 共鸣**：**每片 ≥2 个「这说的就是我」瞬间**——从观众自己的生活取景
     （已读焦虑、周报、被催办、加班…），标出在哪句。共鸣和好笑、涨知识是三种
     不同的留人机制，必须主动设计，不许撞运气
   - **R 节奏（时钟检查）**：把口播稿按时长等分 4 段，每段至少有一个爆点
     （包袱/反转/数据冲击/共鸣点）——影视飓风拉片千万播放视频的结论是
     爆点等分全片，哪段空白哪段就在漏人。空白段 = 补料或砍掉
任何一项不过 = 当场重写，禁止带病进分镜。
**门禁通过即直接往下走一路出片，不向用户要稿件审批（祥瑞 2026-06-12 定）**——
自检机制就是把关人，用户要的是成片不是草稿；成片不对再按反馈返工。
- **时长完全由"讲清楚需要多长"决定，不预设任何档位（祥瑞 2026-06-11 重申）**：
  禁止"概念 1 分钟、复杂 2-3 分钟"这种档位思维——那也是死板。该问的是：把这个内容真正讲明白、讲到位，
  需要几个必讲的点、每个点要铺垫多少？时长是这个判断的自然结果。1 分钟讲不清一个概念就别硬压成 1 分钟，
  该 3 分钟讲透就 3 分钟；30 秒能说清就别拖。红线只有一条：不注水，每句有信息增量。
- 钩子 → 概念锚点 → 讲解（翻译官话术）→ 吐槽/反转 → 结论 + 互动
- **镜头语言（祥瑞 2026-06-11 定，反对"一句一镜"的机械模板）**：
  - **一句话可以多镜头**：一句口播讲一个过程、列举多项、或有视觉转折时，拆成多个镜头（蒙太奇）——
    media 场景多 shot、或一句话拆成连续几个 scene。别死守"一句=一镜"
  - **转场按叙事关系选，不千篇一律**（scene 填 `transition` 字段）：
    并列/递进（同主题继续）→ `cut` 硬切（最常用，干净保节奏，**大部分切换都该硬切**）；
    时间流逝/柔和关联 → `dissolve`；转折反转（"但是"/打脸）→ `fadewhite` 闪白；
    段落结束/情绪沉淀 → `fadeblack` 黑场；空间转换/并列对比 → `slideup`/`wipeleft`；聚焦强调 → `zoomin`。
    **滥用花转场显廉价**——克制，多用硬切，关键转折才上特殊转场
- 写完通读一遍：有没有不口语的书面腔、有没有超过 25 字的长句

### Step 3: 设计 storyboard（内容驱动自由设计，不套模板）
**核心范式（祥瑞 2026-06-11 升级）**：分清「品牌身份」和「模板」——
- **品牌外壳固定 = 身份不是模板**：左上 logo / 右上期数 / 底部字幕条 / 安全区 / 结尾搜索框，
  一眼认出万涂幻象，必须稳定。这层不动。
- **内容呈现完全自由 = 真正的不套模板**：concept_card / diagram / whiteboard 这些固定版式
  **降级为「工具箱 / 参考样式」，不是必选模板**。主路径是 **AI 拿到内容，像设计师一样为每条内容
  现场设计每一屏**——用设计语言（编辑排版 editorial / 手绘 Rough.js / 数据图 ECharts，见
  design-system.md）+ GSAP 动效自由创作，每片每屏都能不同，甚至现场发明新呈现。
  优先 `demo`（现场写 HTML）和 `editorial`（编辑版式）这类自由场景，而非套 concept/diagram 壳。
- 判断标准：这一屏的设计是「从这条内容长出来的」，还是「从类型库里挑的壳」？后者就是模板，返工。
**横版分镜先读 [references/landscape-components.md](references/landscape-components.md)（三支柱横版适配 + 版式骨架五件套）**：
横版的美=同一套万涂幻象设计语言(编辑排版骨架/Rough手绘/开源图表)在 1920×1080 上现场创作,
不是另起炉灶的白卡玻璃章——那是 AI 味。硬底线:每屏主视觉锚点、左对齐编辑感优先、
**产品名出现 logo 必须同屏**、媒体镜带 overlay 标画面身份、**截图二维码必须打码**。
- **图主文辅（祥瑞 2026-06-13 定，硬规则）**：整片以**画面**为主、文字为辅。每个镜头都要有
  图像级主视觉——真实素材/手绘插画/画板风成图(seedream 简笔画卡)/ECharts 图表/Rough 手绘图解/
  常驻角色/logo/截图，且**与这句口播精准对应**（说曲线给曲线，说装它给终端）。
  编号列表、纯文字排版这类"文字型版面"只能当节奏点缀（全片 ≤2 镜），不许撑主体——
  文字是画面的注解，不是画面本身。自检：抽帧逐镜问"这一屏的主角是图还是字？"字多图少 = 返工补视觉。
- **🚦 质量闸门（硬关卡，祥瑞 2026-06-13 定，render.py 强制执行）**：sb.json 写完**必须过 lint**——
  铁律写在文档里没强制力，模型会偷懒套最省事的纯文字版式，所以装了 `scripts/lint_storyboard.py`
  做渲染前关卡：① 纯文字版面（impact_text/editorial/concept_card）> 2 镜 → 拒绝渲染；
  ② 连续 ≥3 镜同一 type → 拒绝渲染；③ 全片 0 个图像级镜头 → 拒绝渲染。
  `render.py` 渲染前自动跑、不过直接退出，**不许加 `--skip-lint` 绕过**（那是排查专用）。
  手动先验：`python3 $SKILL/scripts/lint_storyboard.py sb.json`，看分布 + 判定，红的全改完再进 Step 4。
  闸门只卡下限，过了 lint 不等于做好——图像级现场设计的丰富度仍按本节铁律自检。
- **🎨 手段要多样，别一把梭 demo（祥瑞 2026-06-14 定）**：lint 只看 type 分布过没过，
  **整片清一色 Rough+GSAP demo 框图照样单调**（过了 lint 也会被嫌「skill 没用全」）。
  写分镜时按内容主动在全套工具间换着选最贴切的，别图省事只堆 demo：
  - **seedream 生图**：`seedream-image/gen_image.py`（ARK_API_KEY 在 Vault `00.系统/.env`）生
    黑底白线稿+翠绿点缀（和常驻角色同语言）→ `char_cutout.py` 抠透明 → demo 里 `<img>` 贴 或 media(fit:contain) 整屏
  - **常驻角色立绘**：漫画陪看主角(desk/excited/yawn…)是品牌资产可跨片复用，贴 demo 一侧 + GSAP back.out 弹入
  - **超大动态大字砸场**：200px+ 大字 `gsap.from({scale:3,ease:'power4.in'})` 砸入
  - **media 多 shot / 真实成片 broll** 摆证据
  目标：抽帧看整片，呈现手段是不是「换着来」，而不是一个模子套 16 遍。
**按 [references/storyboard-schema.md](references/storyboard-schema.md) 写 sb.json**。
- **框架是死的，编排是活的**：固定的只有品牌框架和留存五规则；叙事结构、场景类型组合、
  节拍必须按题材重新设计。连续两片结构雷同 = 失败（记忆篇是故事驱动，Token 篇是演示驱动，下一片再换打法）
- **HTML 框内排布纪律（祥瑞 2026-06-11 定）**：demo/concept/diagram/screenshot 这些 HTML 设计场景，
  内容必须在内容窗口框（content_rect）范围内排布好——该显示的完整显示、不溢出窗口、不被白边框遮挡，
  元素间距/字号按框的大小适配。媒体视频素材保持 cover 填满（默认），不做完整展示+模糊铺底。
- **设计图统一用万涂幻象设计语言（去 AI 味，见 [references/design-system.md](references/design-system.md)）**：
  三支柱=编辑排版骨架(editorial 场景)+手绘草图(Rough.js)+开源图表(ECharts/Mermaid)，GSAP 驱动。
  去 AI 味禁区：禁 emoji 当图标、禁居中对称堆卡、禁千篇一律玻璃卡。优先编辑层级+手绘人味。
- **设计图节奏要慢够看清，但禁止人声断档（祥瑞 2026-06-11/12 两次校准）**：
  信息密的设计图要停留够久没错，但**实现方式是配更长的口播，不是用静默画面去等**——
  render.py 已内置"智能剪口播"：画面最多比口播长 `max_gap` 秒（默认 1.0，顶层/单场景可覆盖），
  min_dur 再大也被钳住。所以正确做法：① 一屏 3-4 条的设计 → 口播也写 3-4 句逐条讲到（多句一镜）
  ② demo_js 动画节奏**按口播时长设计**（出场序列在口播结束前 1s 内全部落位），不按 min_dur 想象
  ③ 真需要无声停留的设计性瞬间（钩子定格）单场景给 `max_gap` 但 ≤1.5s。
  VOL.07 教训：min_dur 拉时长攒了 43.6s 人声断档（占全片 1/4），观众听着就是"卡了"。
- **demo 场景每片现写**：`demo` 类型 = 窗口里放一段为本片内容从零写的 HTML 动画
  （概念可视化/过程演示/数字动效），禁止复用往期的 demo HTML，只继承品牌色和确定性 CSS 写法
- 场景类型配比参考 style-guide 第二、三节，但配比是参考不是配方
- **招牌场景不跨片复用（祥瑞 2026-06-11 定，反模板硬规则）**：对比卡(diagram compare)、
  数字滚动 demo、某种 impact 句式这类"顺手就能套"的场景，写分镜时先回顾最近 2-3 片用过什么——
  连续片子套同一个招牌场景 = 模板化 = 返工。VOL.04/05/06 连着套对比卡就是反面教材。
  同一个意思有很多种呈现：对比可以用对比卡，也可以用工位网格熄灭、数据冲击、反转 demo、媒体画面+旁白。换着来
- 每个 scene 先把 image/video 字段留好路径（下一步去填实）

### Step 4: 收集素材（全网开放 + 标注来源）
**素材丰富度铁律（祥瑞 2026-06-11 定，最高优先级）**：每个视频项目都要把素材做到
**尽可能完整、越丰富越好**——这是观感的命门。具体要求：
- **配图必须和内容相符，地点尤其严（祥瑞 2026-06-11 定）**：说苏州就不能配郑州，说北京中关村
  就不能配随便一个城市天际线——张冠李戴比没画面更糟。真实地点/机构画面找不到合规素材时，
  改用文字/数据/地图示意把地点写准（如数据卡写明"苏州工业园"），不强行配不符的通用画面。
  通用氛围(城市/科技/人物剪影)只能用在不指向具体地点的泛叙述里。
- **逐句配画面**：口播稿每一句讲的内容，画面上要有对应的真实素材或设计呈现，
  字幕说 A 画面就给 A，禁止"字幕在讲、画面对不上"。一句列举多项（"论文、数据、会议纪要"）
  就配多个画面快切，一项一个镜头
- **媒体场景多 shot**：能多给画面就多给，单个场景 2-4 个 shot 轮换比一个 shot 干到底丰富得多；
  从源素材（妙记/视频/网页）里穷尽式多截取贴切片段，宁多勿少
- **链接/素材模式**：把源材料里所有能用的画面（视频片段、图表、截图、案例图）都挖出来备选，
  抽帧建候选网格逐一确认内容，再按字幕匹配编排
- **纯设计场景适度**：concept_card/impact_text/diagram 这类无真实画面的设计场景是叙事节奏需要，
  但占比别太高——真实画面太少会显得"素材单薄"，设计场景之间多穿插真实素材
- 自检：渲染后逐场景抽帧，确认每句字幕都有匹配画面；对不上 = 返工补素材

**按 [references/asset-sourcing.md](references/asset-sourcing.md) 决策树执行**：
- 全网图片 → `scripts/fetch_image.py`（人物/事件/新闻图都能用，scene 标 `source: "素材来源：网络"`）
- 图解 → diagram 场景零素材；插画/画板风成图 → seedream；截图 → web-access；broll → fetch_broll.py
- **常驻角色/手绘人物** → seedream 单姿势生成 + `scripts/char_cutout.py` 抠图+完整性自检
  （asset-sourcing.md 第 7 节管线：禁裁设定图、prompt 要求全身完整四周留白、触边告警必 Read 确认）
- 每个素材拿到后**Read 确认内容**再填进 sb.json，禁止盲用；外部素材必填 source（渲染器自动打角标）
- 不搬其他博主的整段成片解说；素材片段引用单源 ≤10s
- **BGM（可选，Step 0 问用户要不要）**：要的话按当片调性选 mood（tech/upbeat/suspense/chill/epic/funny/news，
  `fetch_bgm.py list` 看表），跑 `fetch_bgm.py grab --mood X --out assets/bgm.mp3 --duration <片长>`，
  sb.json 填 `bgm` + `bgm_volume`（默认 0.14）。渲染器自动 ducking：说话时 BGM 让位，停顿浮上来。
  调性判定来自 Step 1 研究：同类爆款视频惯用什么 BGM 风格就选对应 mood。
  **发布提示**：站外免版权曲保成片完整；发抖音/视频号时若站内曲库有热门同风格音乐，
  在平台编辑器替换更好（热门音乐自带流量加成且零版权风险）。
  **双版本自动输出（祥瑞 2026-06-12 定）**：配了 BGM 时 render 自动多产一个 `final_nobgm.mp4`
  无伴奏版——发平台想用站内热门曲就拿无伴奏版去配，两版都在工作目录
- **文档/截图类素材必须卡装（祥瑞 2026-06-12 定）**：PDF 页面、网页截图、聊天记录这类
  "文档感"素材禁止全屏裸铺（边缘生硬难看），scene 标 `"fit": "card"`——自动缩 84% +
  米白细边 + 品牌深底嵌入

### 自实现效果库（剪映式效果用 ffmpeg/CSS 自造，全自动出片，祥瑞 2026-06-11 定）

剪映的特效画面是剪映引擎专属渲染的，CLI 化只能写特效名进草稿、画面搬不进 ffmpeg。
所以方向是**用我们自己的 ffmpeg/CSS 实现剪映式效果**，剪映只当"参考图鉴"。
排版保真（HTML 品牌画面不动），效果是叠加的增强。已实现/规划：
- ✅ **场景智能转场**（render.py，`sb["transitions"]` 默认开）：xfade 自实现叠化/闪白/黑场，
  按场景语义匹配（impact_text=闪白、concept_card=黑场叠、其余=自然叠化），不为配而配；
  转场缩短时间轴后自动重算 SFX 对齐
- ✅ **GSAP 动画引擎**（2026-06-11 接入，jheaven 级动效）：`assets/vendor/gsap.min.js` 已附带，
  record_scene.js 已支持逐帧捕获 GSAP（`gsap.globalTimeline.totalTime(t)` seek，rAF 驱动不走 WAAPI）。
  **demo 场景动画优先用 GSAP** 而非 CSS keyframes：timeline 精确编排出场序列、elastic/back/expo
  专业缓动（真弹性物理回弹，CSS 做不出）、stagger 错落出场一行搞定、数字滚动、SplitText 文字逐字。
  HTML 里 `<script src="file://.../assets/vendor/gsap.min.js"></script>` 引入即可。
  写 GSAP 代码参考已装的 gsap-core / gsap-timeline / gsap-plugins skill（官方知识，按需加载）
- 规划：画面特效（震动/缩放/故障/聚焦）、钩子大字砸场、数字滚动用 GSAP 重做

**分层模式 6 轨**：HTML 设计场景渲染成「无文字纯背景」(bare 模式)，媒体场景用真实素材逐 shot 拆开，

accent 高亮/普惠 Heavy 正体，换来彻底可编辑 + 充分用剪映素材特效。设计哲学：HTML 出剪映做不出的
设计资产，剪映做素材组装+精修，不是互相取代。三轨成品版保真但不可改文字，按需选。
限制：依赖 render.py 的 temp/ 中间产物（出过片才能导）；剪映保存后草稿转加密，
CLI 单向生成不能回读；macOS 剪映无自动导出接口，导出手动点。
**铁律**（都是踩坑换来的，导出器已全部内化）：
① 生成前必退剪映（运行态覆盖草稿目录会损坏草稿，脚本自动退）；
② 主文件名必须 `draft_info.json`（剪映 10.6 找这个，pyJianYingDraft 只写 draft_content.json，脚本自动补）；
③ 素材必须复制进草稿目录 materials/（剪映是沙盒应用，无权访问 ~/Projects，否则"暂无访问权限"，脚本自动复制改路径）；
④ 补 cover 缩略图 + 修 meta tm_duration（否则草稿箱不显示/判损坏）；
⑤ 默认纯三轨，转场/动画是 `--effects` 实验开关；⑥ 生成后手动开剪映。
**关于解密**：剪映 10.6 保存后的草稿是真 AES 加密（base64 解码后熵 7.999），解密要逆向剪映二进制+密钥绑设备，不现实也不必要——我们是生成明文草稿，剪映能读（保存时才加密），单向导出。

### Step 7: 自检（必做，不跳过）
0. **事实校验（一票否决）**：演示动画和口播里出现的每一个数据必须实测或有来源——
   token 切分用 tiktoken 现场跑（`python3 -c "import tiktoken; ..."`，演示卡标注"实测"），
   价格/年份/数字对研究报告来源核对。编造一个数据 = 整片作废（2026-06-10 Token篇教训：
   切分和编号最初是编的，被祥瑞当场质疑）
1. `ffprobe` 看总时长、视频音频双流存在
2. **抽每个场景中点帧 Read 看效果**：文字有没有溢出、图有没有糊/裁坏、高亮位置对不对；**配图与口播是否对得上**(尤其地点：说A配的就得是A，不符=返工)；
   **排版四查（landscape-components.md 自检清单,2026-06-12 定）**：主视觉锚点有没有/
   产品名出现 logo 在不在同屏/字号在不在阶梯上/媒体镜有没有 overlay 或 source 标画面身份；
   **装饰元素纪律**（祥瑞 2026-06-11 定）：短横/分隔线/下划线/角标这些装饰，必须与文字保持
   明确间距、不重合——游离的 absolute 定位最容易压字，优先用 flex gap 把装饰挂在文字下方固定间距
3. `volumedetect` 看 max_volume 在 -1~-6dB 区间
3.5 **静默检查（祥瑞 2026-06-10 定）**：`silencedetect=n=-45dB:d=1.0` 必须 0 命中——
   任何 >1s 的无声段=返工。病根通常是 min_dur 硬拉时长但口播没跟上：
   要么补口播填满（首选，信息密度↑），要么收短 min_dur；
   钩子停留等设计性静默也压在 1s 以内
3.6 **人声断档检查（祥瑞 2026-06-12 定，BGM 会骗过 silencedetect 的洞）**：
   配了 BGM 时 silencedetect 永远 0 命中但人声照样断档。必须对比 manifest 口播时长 vs
   实际场景时长：逐场景算 `gap = 场景dur - 口播dur`，gap>1.2s 的列出来——
   除显式 max_gap 的设计性停留外，命中=返工（补口播或确认钳制生效）。
   render.py 智能剪口播默认兜底（max_gap=1.0），此检查是验证它没被 dur/max_gap 绕过
4. 有问题改 sb.json 对应 scene 重跑 render.py（TTS 没改就不用重跑）
5. **北极星收货（最终一关）**：对照顶部「成片北极星」六条逐条自问并列出结果——
   看完真懂？像节目？包袱长在知识点上？素材句句匹配？信息密度？不像之前任何一片？
   任何一条不达 = 返工对应环节
6. 全部通过才报告"完成"，附成片路径 + 时长 + 场景数 + 北极星六条自查结果
