## 二、文案口播结构（差评君信息密度 × 脱口秀留存）

**目标：有趣（愿意看完）+ 信息密度（真学到东西）。写法学脱口秀，不是堆梗。**

### 脱口秀五技法（每片至少用上 3 个）

| 技法 | 怎么用 | 例 |
|------|--------|----|
| 铺垫-抖包袱 | 先立一个大家默认的常识预期，下一句打碎它 | "我们总觉得大脑是个硬盘…可惜，你大脑的存储协议比网盘还不靠谱" |
| 具体化 | 抽象概念换成具体场景/名词，画面感=笑感 | 不说"日常细节会遗忘"，说"你昨晚吃的是寿司还是面，大脑已经开始编了" |
| 三段式反转 | 列举两个正常项，第三个拐弯 | "硬盘：原样存、不会忘、读一万次都一样。大脑：读一次，改一次" |
| 笑点后挂干货 | 笑完的 2 秒注意力最高，知识点紧跟在 punchline 后面 | 包袱→"这就是心理学说的再固化效应" |
| callback 回扣 | 结尾把开头/中间的梗再抖一次，留存利器 | 开头立"硬盘"梗 → 结尾"你俩的硬盘可能都坏了" |

### 讲解员腔 vs 差评君味（对照例句，写前必看）

**模型写知识内容的默认输出是"讲解员腔"**——定义准确、逻辑清楚、零包袱，像教科书。
这是本 skill 的头号反面模式（2026-06-11 连续三版翻车实证）。同一个信息点，两种写法：

| 讲解员腔（返工） | 差评君味（通过） |
|---|---|
| 聊天 AI，你问一句它答一句，答完就完了。 | 你问它机票怎么买便宜，它哐哐甩你八百字攻略，结尾还贴心来一句"还有什么能帮您？"——帮？票我自己订的，钱我自己付的，你帮什么了？ |
| Agent 会把任务拆成小步骤执行。 | 它接到活第一件事不是干，是列清单——查航班、比价、填信息、付款，跟你妈出门前列购物单一个习惯。 |
| 它有自我纠错能力。 | 最便宜那班卖完了？它不死机也不喊你，自己默默换下一班接着订——这个心理素质，比你强。 |
| 它跟你平时用的聊天 AI 根本不是一个物种。 | 你以为它是升级版 Siri？人家已经不陪聊了，人家上班了。 |

**判别法**：把一句稿子念出来，你会不会下意识停顿等观众笑？不会 = 讲解员腔。
**吐槽的本质**：在事实后面追一刀（评价/类比/拆台），刀越具体越好——
"比你强"不如"这个心理素质比你强"，"很离谱"不如"帮？你帮什么了？"。

### 结构（场景类型对应，2026-06-10 横纵分析修订）

| 段落 | 写法 | 场景类型 |
|------|------|---------|
| **开头钩子（0-4 秒）** | **media 场景：全片最强视频素材满窗 + hook_text 大字砸屏 + 反常识断言口播**。禁止静态字卡开场、禁止自我介绍、禁止品牌前置 | media + hook_text |
| 概念锚点 | 给一个记忆点词（标题 2-6 字），紧跟钩子之后 | concept_card |
| 主体讲解 | 铺垫→包袱→干货，每 2-3 句一个新信息点 | whiteboard / diagram 交替 |
| 吐槽/反转 | punchline 一句话砸出来 | impact_text |
| 结尾 | callback 回扣 + 互动问题（评论钩子） | ending |

### 留存五规则（前 3 秒完播权重 40%，违反任何一条都在漏人）

1. **3 秒冲突**：第一句必须是反常识断言/冲突/悬念裂口（"全世界几亿人记得一个活人的葬礼"），最猛信息放最前，背景铺垫后置
2. **3-4 秒视觉重置**：任何素材在窗口里停留 ≤4 秒（多 shot 切换/图解逐条出现/punch-in 都算重置）
3. **开环不闭合**："答案可能吓到你"类悬念中段埋下、结尾才收，强制完播
4. **进度暗示**：60-70% 处口播给"最后一关/最关键的来了"信号（顶部进度条已自动提供视觉版）
5. **循环结尾 + 评论钩子**：结尾 callback 开头的梗（语义循环），最后一句是激发性问题

- 口语短句，单句 ≤ 25 字；narration 是完整可朗读的话
- 语速 1.1x ≈ 320-360 字/分钟；竖屏 60-90 秒 ≈ 350-550 字
- 一句话 = 一个镜头 = 一个 scene，平均 4-7 秒一切；每 10-15 秒必须有一个笑点或惊点（留存钩）
- 自检：通读一遍，如果某 15 秒里既没包袱也没新知识点，砍掉或重写

## 三、场景配比（竖屏 60-90 秒参考）

| 类型 | 数量 | 用途 |
|------|------|------|
| concept_card | 1 开头 | 墨绿渐变 + 超大期数数字 + 大白标题 |
| whiteboard | 2-3 | 白底点阵 + 白卡包图 + 粉桃 caption 胶囊 + 大灰水印 |
| **diagram** | 2-3 | **讲解主力**：flow 流程卡 / compare 对比卡 / list 要点卡，全自动生成不需要找图 |
| screenshot | 0-2 | 证据感：自己截的图，白卡倾斜 + 粉莓荧光高亮 |
| broll | 0-1 | 仅限：官方发布素材 / 自己录屏 / AI 生成视频 |
| impact_text | 1-2 | 深底光斑 + 白字粉莓描边砸字 |
| ending | 1 结尾 | 品牌结尾卡（VANTASMA + 万涂幻象 + 绿光互动框） |

**素材纪律（红线）**：
- ❌ **禁止使用其他博主/UP 主成片里的画面**（截帧、片段都不行）
- ❌ 禁止连续 3 个同类型场景
- ✅ 图解优先 diagram 自动生成；插画用 seedream 自己生成；截图截自己能访问的页面
- ✅ 信息"更容易理解"靠 diagram：能画流程就别堆字，能对比就上 compare

## 四、各场景用好它的要点

- **concept_card**：title 2-6 字；subtitle 英文全大写；number 默认 "01"（可当期数）
- **whiteboard**：image 必给（seedream 生成的白底插画最搭）；watermark 2-4 字；caption ≤ 12 字结论
- **diagram**：flow 3-4 步（t=2-4字 d=≤14字）；compare 左右各 3-4 点（每点 ≤ 10 字）；list 2-4 条
- **screenshot**：highlight [x0,y0,x1,y1] 相对坐标；只用自己截的图
- **impact_text**：text ≤ 8 字；narration 同步说出这句
- **花字 keywords**：每片 2-4 个，at = 字序号 ÷ 5.5 估秒
- **motion**：in 缓推（默认）/ out 揭晓 / up 长图上移

## 五、音效与 BGM

| 音效 | 时机 |
|------|------|
| impact | concept_card 开场、重磅结论 |
| ding | 花字/数据/impact_text |
| whoosh | whiteboard/diagram 切入 |
| pop | 轻元素、ending |
| tick | 普通切镜可不加 |

每场景最多 1 个 sfx。BGM 可选（NCS 免版权，音量 0.13），不加也成立。

## 六、已知限制

- ASS 动画文字（花字/砸字/字幕）不要 emoji；HTML 卡片里可以用
- diagram 文案过长会撑破卡片，遵守上面字数上限
- broll 源视频自带字幕/水印的不要用
